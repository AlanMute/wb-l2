Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
Интерфейс равен nil из-за того, что одно из полей интерфейса в языке Go представляет собой ошибки и имеет значение nil. Конкретно, это поле представлено в виде (таблица интерфейса)

type iface struct { 
	tab *itab 
	data unsafe.Pointer 
} 

Тип error является интерфейсным типом, и вся информация о его методах, типах и прочем скрыта в tab, известной как интерфейсная таблица. Однако, после присвоения этого интерфейса переменной err, поле data перестает быть nil и получает указатель на конкретный тип *os.PathError. Это приводит к тому, что при сравнении с nil результат оказывается false. Вместе с тем, из-за отсутствия значений в интерфейсной таблице при выводе переменной значение выводится как nil.
...
Output
<nil>
false
```
